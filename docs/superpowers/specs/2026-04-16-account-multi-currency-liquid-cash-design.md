# 持仓账户多币种流动现金设计

## 背景

当前每个持仓账户只有一个数值型 `liquidCash` 字段，默认跟随账户主币种。这个结构无法表达同一个持仓账户下同时存在 USD、HKD、CNY 等多笔流动现金，也不方便在界面中新增、删除或调整单笔现金记录。

现有账户详情页虽然把流动现金展示在持仓列表之后，但编辑页中“流动现金”区域后面仍然有“新增股票/基金”区块，因此它并不是真正固定在账户卡片最底部。

## 目标

- 每个持仓账户支持多笔流动现金记录
- 每笔流动现金可以单独选择币种
- 账户总市值继续显示账户主币种
- 多币种流动现金始终计入账户总值与总资产
- 流动现金区域在持仓账户卡片中始终位于最底部
- 保持现有本地存储数据可自动迁移，不要求用户手动清数据

## 非目标

- 不改“现金仓位（仅备忘）”的语义与位置
- 不改独立“现金账户”的数据结构
- 不新增 USD、HKD、CNY 之外的新币种
- 不调整投机账户是否计入总资产的规则

## 方案对比

### 方案 A：对象映射

将 `liquidCash` 改为类似 `{ USD: 100, HKD: 200 }` 的对象。

优点：
- 计算逻辑简单
- 改动相对较小

缺点：
- 不适合界面逐条编辑
- 难以支持逐条删除、排序或未来增加备注
- 空值和币种键的维护容易混乱

### 方案 B：数组记录（推荐）

将 `liquidCash` 升级为 `liquidCashItems` 数组，每条记录形如：

```json
{
  "currency": "USD",
  "balance": 120.5
}
```

优点：
- 自然支持多币种、多条记录、逐条新增删除
- 最适合当前 UI 交互
- 未来可扩展备注、来源等字段

缺点：
- 需要在加载旧数据时做一次迁移

### 方案 C：并入 holdings

把流动现金也作为一种特殊持仓并入 `holdings`。

优点：
- 渲染结构统一

缺点：
- 现金与证券/基金语义混淆
- 编辑与计算逻辑会变得更绕
- 容易影响现有持仓展示和价格计算逻辑

## 决策

采用方案 B：为每个账户引入 `liquidCashItems` 数组，替代单值 `liquidCash`。

## 数据结构

### 新结构

```json
{
  "name": "大道至简",
  "platform": "嘉信",
  "currency": "USD",
  "type": "stock",
  "liquidCashItems": [
    { "currency": "USD", "balance": 100 },
    { "currency": "HKD", "balance": 500 }
  ],
  "holdings": []
}
```

### 兼容旧数据

应用加载本地数据或导入 JSON 时，对每个账户执行以下迁移规则：

1. 若已存在 `liquidCashItems`，直接使用
2. 若不存在 `liquidCashItems` 但存在旧字段 `liquidCash`
   - 当 `liquidCash > 0` 时，迁移为一条 `{ currency: account.currency, balance: liquidCash }`
   - 当 `liquidCash <= 0` 时，迁移为空数组
3. 迁移完成后保留运行态只依赖 `liquidCashItems`

这样可以兼容旧的 localStorage 数据和旧格式导入文件。

## 计算逻辑

### 账户总值

账户总值由两部分组成：

1. 持仓市值
   - 股票账户：`shares * 实时价格`
   - 基金账户：`marketValue`
2. 多币种流动现金总值
   - 每笔 `liquidCashItems` 先换算为 CNY
   - 再在需要显示账户主币种总额时，从 CNY 反算回账户主币种

### 显示规则

- 账户卡片顶部“总市值”继续显示账户主币种
- “折合 CNY”继续展示人民币等值
- 展开详情时，流动现金明细按原币种逐条显示
- 总资产与历史快照继续以 CNY 汇总，不改变统计口径

### 汇率处理

继续复用现有三币种汇率：

- `USD -> CNY`
- `HKD -> CNY`
- `CNY -> CNY`

新增一个“从 CNY 反算到目标币种”的辅助换算函数，避免账户主币种显示错误。

## 界面设计

### 账户页

- 展开账户详情后，先展示 `holdings`
- 然后展示“流动现金”区域
- “流动现金”区域固定放在账户卡片内容最底部
- 多笔流动现金逐条展示：
  - 币种
  - 原币种金额
  - 可选地显示折合主币种或折合 CNY 作为辅助信息

若账户没有任何流动现金记录，仍展示一个空状态，例如“暂无流动现金”。

### 编辑页

- 每个账户卡片内顺序调整为：
  1. 账户元信息编辑区
  2. 持仓列表
  3. 新增持仓区
  4. 流动现金编辑区（固定在最底部）
- 流动现金编辑区支持：
  - 逐条修改币种
  - 逐条修改余额
  - 删除单条记录
  - 新增一条空记录

这样“流动现金始终在持仓账户的最底部”的要求在编辑页和账户页都成立。

## 状态与交互

- 新建账户时，默认 `liquidCashItems: []`
- 新增流动现金记录时，默认币种可设为账户主币种，余额为 0
- 删除最后一条流动现金后，保留空数组，不回退到旧字段
- 导入旧数据时在内存中做迁移，再进入正常渲染和保存流程

## 影响范围

- 默认数据中的账户结构
- 账户总值计算函数
- 概览页总资产计算
- 账户页展示
- 编辑页流动现金编辑逻辑
- 历史快照生成
- 本地存储加载与 JSON 导入的兼容迁移

## 错误处理

- 缺失汇率时继续使用现有默认汇率兜底
- 非法数值输入继续按现有逻辑回落为 `0`
- 导入文件若缺失 `liquidCashItems`，自动按旧字段迁移
- 导入文件若两者都缺失，视为无流动现金

## 测试要点

- 旧数据 `liquidCash` 能自动迁移为 `liquidCashItems`
- 同一账户多币种流动现金会正确计入账户总值
- 账户总值主币种显示正确
- 总资产和月度快照数值正确
- 编辑页可新增、修改、删除多笔流动现金
- 账户页与编辑页中，流动现金都位于账户卡片最底部
